## 关于gost
  gost是一款存放于GitHub的开源项目，3.0使用其自营的网站 [^1]

## 简单的gost使用
### 基本使用
```markdown
gost -L http://:8080
```
```markdown
gost -L http://:8080 -F http://192.168.1.1:8080
```
### 关于代理转发和通道
[文章原文](https://gost.run/concepts/proxy/)

我的理解是使用一个gost首先需要监听某个端口，然后对接收的进行一些处理和转发的过程。

### 对监听的理解
监听的使用即在gost加一个-L参数并写对应的处理方式，已知的处理方式有http,https,socks和加密隧道协议等监听，同时，亦可以使用tcp再监听的同时完成转发（-L tcp://:8080/192.168.1.1:80）。

### 对转发的理解
通常的转发使用-F参数，在-L监听之后。同样的。转发亦可以使用http，https，socks或加密隧道协议完成转发工作。

### 代理链
结合监听与转发，我们就可以设计一些复杂的代理链。

我设计过一个兼顾加密隧道，和网页伪装的隧道模式（可以欺骗cdn让你的其他内容通过443进站
）

服务端（即在服务器跑的gost）
```markdown
gost -L ss+mwss://加密协议:密码@:443？cert=证书路径&key=私钥路径&bypass=127.0.0.1
```
客户端（即跑在你自己电脑的gost）

```markdown
gost -L socks5://:1080 -F ss+mwss://加密协议:密码@服务器IP:443 

```
这样，你就可以在比如clash等代理中让流量从1080端口进入，由你的服务器转发，同时，你也可以使用“ssh -p 1080 账户@域名”以实现在以对外仅开发443端口的服务器中实现ssh功能

### 实用建议
假如我们和服务器已经存在有隧道（比如Cloudflare提供的mesh组网，参照Cloudflare的文章），我们即可通过内网的IP和仅使用socks进行转发以减低我们的延时和提高传输速度。

此时服务端
```markdown
gost -L socks5://:1080
```
并在clash等代理中指定服务器的内网地址和socks方式即可完成转发工作
